Some replies to my friends' consult about Hyperledger

Posted on 2018-10-11

Frirstly Hyperledger is a combination of many projects. Here we discuss the most popular one–Hyperledger Fabric.

According to my experience, first its tps(transaction per seconds) is not very high. In practical it usually can deal several hundreds transactions per second,which is not fast enough under many scenes.

Second, it did not have a good scalability. This is actually similar with the first point. If it has a good scalability then tps will be not a problem here, because with high scalability we just add more computers can meet the requirement.

Third, it is not that decentralized as other project, for it use kafka as consensus mechanism, it is vulnerable when one endorse node is malicious. And if you use kafka cluster, the whole system become much more centralized and rely heavily on the cluster, which maybe somewhat against the purpose of using blockchain. Fourth is the config is complex in many ways. This point also make the third point more serious, because many fresh man will deploy the blockchain centralized to reduce the complexity. In my view a centralized blockchain is useless and much worse than a traditional database.
