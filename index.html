For persons who are familar with C language, there are so many pitfalls hidden in python. Let’s see the example below:

is

rather than

[[True, False, False], [False, False, False], [False, False, False], [False, False, False], [False, False, False]]

The reason is that * operator for object in python means shallow copy, and list * n means n shallow copies of this list. As we know, shallow copy only copy the object itself, will not copy any object inside this object. Here [False] * 3 is a list object [False, False, False] in object [[False, False, False]]. When we apply *5 on it, it will only copy the outer list object, the inner object will remain the same address as before. As a result, we we assign True to books[0][0], the same objet will be changed.

Solution

My basic thought is that I use two cursors to record buy index and sell index. We traverse the entire prices array, every time when the j + 1 one bigger than the j one, which means the sequential is increased, we increase j cursor. When the j + 1 one is less or equal than the j one, we calculate the profit we make in this period and add it to the toal profit, then assign i to j + 1 so that we can move on to next peroid. One edge case is that the sequence has no decline period. To sovle this case we should do an extra judge that if until the last one the sequence is keep increasing, we calculate the last period’s profit and add into total profit. (This logical maybe can be simplified by we add a 0 in the tail of prices array in advance so that we can make the whole judge process uniform.)

One simpler and more brilliant solution can be illustrated by the pic below:

From the picture, we can see that

Which means if you want to get profit D because it is the max proft we can get between day 3 and day 7, there is no difference from summing up every positive differential in that period.

So, we can find that everytime when the j + 1 is one bigger than the j one, we simply get the differential between them and add it to the total profit, after traversing the entire array we can get a decent total profit.

The code is given below:

class Solution:
    def maxProfit(self, prices):
        """
        :type prices: List[int]
        :rtype: int
        """
        ans = 0
        for i in range(1, len(prices)):
            if prices[i - 1] < prices[i]:
                ans += (prices[i] - prices[i-1])
        return ans

0x0 Transaction mined but execution failed

Posted on 2018-08-07

Some times it will occur “0x0 Transaction mined but execution failed” error when you execute contract function on Ethereum. If your contract is tested fully, the main reason should be you use a low gas limit for this function. For example, the metamsk will set gas limit automatically for you according to gas estimation, but if some dynamic parameters in your function changed (such as a function include a loop an d the loop count is decided by time or others’ input), the gas estimation will not output correct result, so the function may fail.

Using a larger gas limit can solve this. If it still not work, write test cases to check if your function works right.

Solution

这里简单的取最低价格，然后记录基于该价格的最大利润。如果有更低价格则记录基于更低价格（将之前的最低价格直接替换掉）的最大利润，并与之前价格的最大利润相比较，取较大的利润。
