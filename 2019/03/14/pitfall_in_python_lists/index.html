Pitfall in Python Lists

Posted on 2019-03-14

For persons who are familar with C language, there are so many pitfalls hidden in python. Let’s see the example below:

It is somehow unintuitive the result of the code above
is

[[True, False, False], [True, False, False], [True, False, False], [True, False, False], [True, False, False]]

rather than

[[True, False, False], [False, False, False], [False, False, False], [False, False, False], [False, False, False]]

The reason is that * operator for object in python means shallow copy, and list * n means n shallow copies of this list. As we know, shallow copy only copy the object itself, will not copy any object inside this object. Here [False] * 3 is a list object [False, False, False] in object [[False, False, False]]. When we apply *5 on it, it will only copy the outer list object, the inner object will remain the same address as before. As a result, we we assign True to books[0][0], the same objet will be changed.
